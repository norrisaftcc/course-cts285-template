## 5.5 API Integration Points

**Where Design Touches Code: UI Components and Data Requirements**

BudgetBoss interface components consume data from backend APIs. Understanding these integration points helps designers create realistic, implementable interfaces.

### Dashboard Components and Data Requirements

**Dashboard:** Primary landing page showing financial snapshot.

**Components:**

1. **Account Balance Summary**
   - **Data source:** `GET /api/accounts/summary`
   - **Response:** `{ total_balance: 5432.18, accounts: [...], net_worth: 4850.00 }`
   - **Update frequency:** Real-time on transaction, hourly background refresh
   - **Design considerations:** Loading state, balance animation, multiple account display

2. **Income This Month**
   - **Data source:** `GET /api/income/current-month`
   - **Response:** `{ total_income: 4200.00, by_source: [...], comparison_to_last_month: +8.5 }`
   - **Update frequency:** Real-time when income recorded
   - **Design considerations:** Income source breakdown, trend indicator, celebration for high-earning months

3. **Tax Fund Status**
   - **Data source:** `GET /api/tax-fund/status`
   - **Response:** `{ saved_amount: 1250.00, estimated_owed_q1: 1500.00, next_deadline: "2026-04-15", days_remaining: 28 }`
   - **Update frequency:** Real-time on transactions affecting tax fund
   - **Design considerations:** Progress bar, deadline countdown, reassuring visual language

4. **Recent Transactions**
   - **Data source:** `GET /api/transactions?limit=10&sort=desc`
   - **Response:** Array of transactions with `{ id, date, amount, description, category, income_source, is_business_expense }`
   - **Update frequency:** Real-time on new transaction
   - **Design considerations:** List vs. card layout, income source badges, business expense indicators

5. **Budget Progress**
   - **Data source:** `GET /api/budgets/current-month`
   - **Response:** Array of budgets with `{ category, limit, spent, percentage, status }`
   - **Update frequency:** Real-time on transaction affecting budgeted category
   - **Design considerations:** Multiple budget bars, color-coded thresholds, encouraging language

6. **Savings Goals**
   - **Data source:** `GET /api/goals/active`
   - **Response:** Array of goals with `{ name, target_amount, current_amount, percentage, deadline }`
   - **Update frequency:** When contributions made or goals updated
   - **Design considerations:** Visual progress indicators, milestone celebrations, deadline display

**Loading states:** All dashboard components must handle:
- Initial load (skeleton screens)
- Empty state (no data yet: "Add your first transaction to get started!")
- Error state (API failure: "We're having trouble loading your data. Tap to retry.")
